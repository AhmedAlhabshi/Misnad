نفّذ **Milestone 4.4A فقط**.

المطلوب هو **ربط خدمة تحليل العقود** التي بُنيت في Milestone 4.3 مع Route الحالي `analyze-contract`.

## المطلوب

بعد نجاح الخطوات الحالية:

1. رفع PDF.
2. استخراج النص.
3. إخفاء البيانات الحساسة.

قم باستدعاء:

```ts
analyzeContract(maskedText, contractType)
```

ثم أعد نتيجة التحليل ضمن استجابة الـ API.

---

## Contract Type مؤقتًا

في هذه المرحلة **لا يوجد اختيار من المستخدم**.

لذلك استخدم قيمة مؤقتة ثابتة:

```text
other
```

ولا تضف أي اختيار في الـ Frontend.

ولا تطلب `userSelectedContractType`.

سيتم تنفيذ اختيار نوع العقد في مرحلة لاحقة.

---

## استجابة الـ API

لا تحذف أي شيء موجود حاليًا.

استجابة الـ API يجب أن تستمر في احتواء:

* success
* message
* textLength
* maskedTextPreview
* piiStatistics

ثم أضف حقلًا جديدًا فقط:

```json
{
  "analysis": { ... }
}
```

إذا نجح تحليل Gemini.

أما إذا فشل تحليل Gemini:

* لا تجعل Route ينهار.
* أعد البيانات الحالية كما هي.
* وأضف:

```json
{
  "analysis": null,
  "analysisError": "..."
}
```

برسالة خطأ آمنة لا تحتوي على نص العقد ولا مفتاح API.

---

## الأخطاء

إذا لم يوجد:

* GEMINI_API_KEY

أو فشل الاتصال

أو فشل التحقق من الـ Schema

فيجب أن يستمر Route بالعمل ويعيد:

* نجاح استخراج النص.
* نجاح PII Masking.
* مع `analysis = null`.

أي أن تحليل AI يعتبر Feature إضافية وليس سببًا لفشل رفع الملف بالكامل.
